Key Architectural Decisions

React Frontend

Modern SPA with component-based architecture. Fast, responsive, real-time updates via WebSockets.

FastAPI Backend

High-performance Python API framework. Async support, automatic API docs (OpenAPI), Pydantic validation.

PostgreSQL

Production-grade relational database. ACID compliance, complex queries, indexing for performance.

Redis

In-memory caching and job queue. Speeds up repeated queries, powers Celery background tasks.

Celery

Distributed task queue. Handles batch processing, scheduled jobs, async operations.

JWT Authentication

Stateless token-based auth. Secure, scalable, supports role-based access control (RBAC).

Coupa API Integration

API Client Architecture

# src/coupa_api/client.py import httpx from typing import List, Optional from tenacity import retry, stop_after_attempt, wait_exponential class CoupaAPIClient: """ Async client for Coupa Procurement API. Handles authentication, rate limiting, retries, and error handling. """ def __init__(self, api_url: str, api_key: str): self.api_url = api_url self.headers = { "X-COUPA-API-KEY": api_key, "Accept": "application/json", "Content-Type": "application/json" } self.client = httpx.AsyncClient( timeout=30.0, limits=httpx.Limits(max_connections=10) ) @retry( stop=stop_after_attempt(3), wait=wait_exponential(multiplier=1, min=2, max=10) ) async def get_invoices( self, start_date: Optional[str] = None, end_date: Optional[str] = None, status: Optional[str] = None, limit: int = 100, offset: int = 0 ) -> List[dict]: """Fetch invoices from Coupa with pagination.""" params = { "limit": limit, "offset": offset, "status[in][]": status or "pending_approval,approved" } if start_date: params["created_at[gt_or_eq]"] = start_date if end_date: params["created_at[lt_or_eq]"] = end_date response = await self.client.get( f"{self.api_url}/invoices", headers=self.headers, params=params ) response.raise_for_status() return response.json() async def update_invoice_activity_code( self, invoice_id: str, line_number: int, activity_code: str ) -> dict: """Update activity code on invoice line in Coupa.""" payload = { "invoice_lines": [ { "line_num": line_number, "activity_code": activity_code } ] } response = await self.client.put( f"{self.api_url}/invoices/{invoice_id}", headers=self.headers, json=payload ) response.raise_for_status() return response.json() async def get_suppliers(self, query: Optional[str] = None) -> List[dict]: """Fetch suppliers from Coupa.""" params = {"limit": 1000} if query: params["name[like]"] = query response = await self.client.get( f"{self.api_url}/suppliers", headers=self.headers, params=params ) response.raise_for_status() return response.json()

API Endpoints (Backend)

GET /api/invoices
List invoices from Coupa with filters (date range, status, supplier). Supports pagination, search, sorting. Returns enriched data with classification status.
POST /api/classify
Classify a single invoice line. Runs classification engine, returns activity code with confidence score, method, and suggestions. Stores result in classification_history.
POST /api/batch-classify
Batch classify multiple invoices (background job). Queues Celery task, returns job_id for polling. Processes invoices asynchronously, sends email when complete.
GET /api/suggestions/:invoice_id/:line_number
Get top 3 classification suggestions for specific invoice line. Includes rationale, confidence scores, and alternative codes.
POST /api/approve
Approve classification (admin action). Updates supplier_activity_mappings, resolves admin_alert, optionally syncs back to Coupa.
GET /api/alerts
List pending admin alerts (non-IT vendors, ambiguous cases, high-value invoices). Filter by status, priority, date.
POST /api/feedback
Submit manual correction (feedback loop). Stores in manual_overrides, updates supplier mappings if pattern detected (3+ corrections).
PUT /api/sync-to-coupa/:invoice_id
Sync classified activity code back to Coupa API. Updates invoice line in Coupa with assigned code. Handles API rate limits and retries.
GET /api/reports/accuracy
Generate accuracy report. Shows auto-assignment rate, manual correction rate, confidence distribution, improvement over time.

Frontend Features

User Interface Screens

📊 Dashboard

Real-time statistics: auto-assigned count, needs review, admin alerts. Charts showing confidence distribution, top suppliers, activity code breakdown.

📋 Invoice Review Table

Interactive table with all unclassified invoices. Filter by confidence, status, date. Search by supplier, invoice #. Sort by any column. Bulk actions.

🔍 Classification Suggestions

Modal showing top 3 suggestions with rationale. Visual confidence indicators. One-click approval or manual selection.

⚠️ Admin Alert Queue

Dedicated view for non-IT vendors and ambiguous cases. Approve/reject workflow. Bulk approval for similar cases.

🏢 Supplier Management

View and edit supplier→activity code mappings. Add new suppliers manually. View classification history per supplier.

📝 Activity Code Editor

CRUD operations on activity codes. Manage keywords for matching. Edit supplier name lists. Import/export CSV.

📈 Reporting & Analytics

Accuracy over time, spend by activity code, top suppliers, manual correction trends. Export to Excel/PDF.

👥 User Management

Add/remove users, assign roles (admin, reviewer, read-only). Audit log of user actions. SSO integration (SAML/OAuth).

🔔 Notifications

In-app notifications for new alerts, batch job completion, approval requests. Email digests for pending reviews.

Sample API Route Implementation

# api/routes/classification.py from fastapi import APIRouter, Depends, HTTPException, BackgroundTasks from sqlalchemy.orm import Session from typing import List from classification.engine import ClassificationEngine from classification.models import ClassificationResult, InvoiceLine from coupa_api.client import CoupaAPIClient from api.dependencies import get_db, get_current_user, get_coupa_client router = APIRouter(prefix="/api", tags=["classification"]) @router.post("/classify", response_model=ClassificationResult) async def classify_invoice_line( invoice_id: str, line_number: int, sync_to_coupa: bool = False, db: Session = Depends(get_db), coupa_client: CoupaAPIClient = Depends(get_coupa_client), current_user = Depends(get_current_user) ): """ Classify a single invoice line. - Fetches invoice from Coupa API - Runs classification engine - Stores result in database - Optionally syncs back to Coupa """ # Fetch invoice from Coupa invoice = await coupa_client.get_invoice(invoice_id) if not invoice: raise HTTPException(status_code=404, detail="Invoice not found") line = invoice.lines[line_number - 1] invoice_line = InvoiceLine( invoice_id=invoice_id, line_number=line_number, supplier=line.supplier, commodity=line.commodity, description=line.description, total_amount=line.amount, currency=line.currency ) # Run classification engine = ClassificationEngine(db) result = engine.classify(invoice_line) # Store in database db.add_classification_result(result, user_id=current_user.id) # Sync back to Coupa if requested and confidence is high if sync_to_coupa and result.confidence >= 70: await coupa_client.update_invoice_activity_code( invoice_id, line_number, result.activity_code ) return result @router.post("/batch-classify") async def batch_classify( start_date: str, end_date: str, background_tasks: BackgroundTasks, current_user = Depends(get_current_user) ): """ Queue batch classification job. Returns job_id for polling status. """ from tasks.classification import batch_classify_task job = batch_classify_task.delay( start_date=start_date, end_date=end_date, user_id=current_user.id ) return { "job_id": job.id, "status": "queued", "message": "Batch classification job queued" }

Pros & Cons

✓ Advantages

  • Real-Time: Instant classification via web UI
  • User-Friendly: Modern, intuitive interface
  • Collaboration: Multi-user, role-based access
  • Scalable: Handle high concurrent users
  • Audit Trail: Complete history with user attribution
  • Direct Integration: Two-way sync with Coupa API
  • Analytics: Rich reporting and dashboards
  • Notifications: Alerts for pending approvals
  • SSO Ready: Enterprise auth integration
  • Mobile Responsive: Works on tablets/phones

✗ Challenges

  • Development Time: 8-10 weeks (vs. 5 weeks for CLI)
  • Infrastructure Cost: Servers, database, caching
  • Maintenance: Ongoing updates, security patches
  • Complexity: More components to monitor/debug
  • Learning Curve: Users need training on web UI
  • Dependencies: Frontend + backend + DB + cache
  • Security: Auth, RBAC, API security required
  • Coupa API Limits: Rate limiting, API changes

Development Timeline

Phase Duration Deliverables
Week 1-2: Backend Core 2 weeks FastAPI setup, database schema, Coupa API client, authentication
Week 3-4: Classification API 2 weeks Classification endpoints, background jobs (Celery), API tests
Week 5-6: Frontend Foundation 2 weeks React setup, routing, auth flow, API client, base components
Week 7-8: Core UI Screens 2 weeks Dashboard, invoice table, suggestion modal, admin alerts
Week 9: Admin Features 1 week Supplier management, activity code editor, user management
Week 10: Testing & Polish 1 week E2E tests, bug fixes, performance optimization, UAT

Total: 10 weeks (2 backend developers + 1 frontend developer)

Recommendation: Hybrid Approach

Start with CLI/Batch (5 weeks) to deliver immediate value, then expand to Web App (10 additional weeks) based on user feedback and adoption.

This phased approach:

  • Validates classification engine before investing in web UI
  • Delivers ROI sooner (CLI can process historical data immediately)
  • Allows user testing to inform web UI design
  • Spreads development cost over time
  • Reduces risk (can stop after CLI if web app not needed)
