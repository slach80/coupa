Getting Started

For Finance Team: Reviewing Classifications

After the system processes invoices, you'll receive an Excel review report with three key sheets:

✓ Sheet 1: Auto-Assigned (≥70% Confidence)

Invoices that the system classified with high confidence. These codes have been automatically applied.

Action: Spot-check for accuracy. If you find errors, use the admin review tool to correct them.

? Sheet 2: Needs Review (20-70% Confidence)

Invoices where the system has moderate confidence. Shows top 3 suggested codes with rationale.

Action: Review suggestions and select the correct code. Your choice will be recorded and improve future classifications.

⚠ Sheet 3: Admin Alerts (Non-IT Vendors)

Vendors flagged as non-IT (cleaning, benefits, legal, etc.). System has suggested a code but needs admin approval.

Action: Verify the suggested code is correct. If approved, the vendor will be auto-classified in future.

📋 Sheet 4: Suggestions (Alternative Codes)

Additional suggestions for invoices in the 20-70% confidence band.

Action: Use these alternatives if the top suggestion doesn't fit.

Running the Classification System

Option 1: CLI Script (Manual Run)

# Process an Excel file
python scripts/classify_invoices.py --input SAMPLE.xlsx --output classified_output.xlsx

# Generate review report only (don't modify input file)
python scripts/classify_invoices.py --input SAMPLE.xlsx --report-only

# Process specific invoice sheet
python scripts/classify_invoices.py --input SAMPLE.xlsx --sheet "Nov2025_Inv"

# Set custom confidence threshold
python scripts/classify_invoices.py --input SAMPLE.xlsx --threshold 80

Option 2: Scheduled Batch Job (Automated)

If configured, the system runs automatically on a schedule (e.g., nightly at 2 AM). You'll receive an email with:

  • Summary of invoices processed
  • Count of auto-assigned vs. needs review
  • Link to review report
  • Admin alerts requiring attention

Option 3: MCP Tools (Real-Time)

If using Claude Desktop or MCP-enabled tools, you can classify invoices in real-time:

# Classify a single invoice
coupa_classify_invoice(invoice_id="12345", line_number=1)

# Batch classify invoices from date range
coupa_batch_classify_invoices(start_date="2025-11-01", end_date="2025-11-30")

# Get admin alerts
coupa_get_admin_alerts(status="pending")

Making Corrections (Feedback Loop)

When you correct a classification, the system learns from it. Here's how:

Method 1: Edit Review Report Directly

  1. Open the review report Excel file
  2. Find the invoice in "Needs Review" or "Admin Alerts" sheet
  3. Enter the correct activity code in the "Corrected_Code" column
  4. Save the file
  5. Run the admin review script:
python scripts/admin_review.py --import review_report_20260501.xlsx

Method 2: Use Admin CLI Tool

# Correct a single invoice
python scripts/admin_review.py --invoice-id "12345" --line 1 --code "WP100" --reason "This is laptop purchase, not software"

# Approve non-IT vendor suggestion
python scripts/admin_review.py --approve-alert 42

# List pending alerts
python scripts/admin_review.py --list-alerts

What Happens After Correction?

  • The correction is stored in the manual_overrides table
  • If the same supplier appears 3+ times with the same corrected code, it's added to supplier_activity_mappings
  • Future invoices from that supplier will be auto-assigned with high confidence (95%)
  • You can track improvement by re-running the classification on the same data

Understanding Confidence Scores

Confidence Range Meaning Action
90-100% Exact historical match or manual override ✓ Highly reliable, auto-assigned
70-89% Strong fuzzy match or commodity match ✓ Good confidence, auto-assigned
50-69% Moderate match, some ambiguity ? Review suggestions, select best
20-49% Weak match or non-IT vendor ? Manual review required
0-19% No match found, new supplier ✗ Manual classification required

Common Scenarios

Scenario 1: Supplier sells multiple product types

Example: CDW DIRECT LLC appears with both WP100 (laptops) and WP320 (VPN licenses)

Solution: System uses commodity/description to differentiate. If uncertain, it will suggest both and ask you to choose.

Scenario 2: New supplier never seen before

Example: "ACME SOFTWARE INC" - no historical data

Solution: System checks commodity/description for keywords. If it finds "software license" → suggests PL150. You confirm, and future ACME invoices auto-assign.

Scenario 3: Non-IT vendor (e.g., cleaning service)

Example: "JAN-PRO CLEANING SERVICES"

Solution: System detects "CLEANING" keyword, suggests SC500 (Property & Facility), flags for admin approval. You approve, and future JAN-PRO invoices auto-assign SC500.

Scenario 4: Supplier name typo or variation

Example: "MICROSOFT (AZURE)" vs. "MICROSOFT AZURE" vs. "MS AZURE"

Solution: Fuzzy matching handles variations. All three will match to the same activity code (IN100) with high confidence.

Troubleshooting

Issue: Classification seems wrong

Check: Review the "Classification_Method" column in the report. This shows why the system chose that code (fuzzy match, commodity match, etc.)

Fix: Correct the code using the admin review tool. The system will learn from your correction.

Issue: Too many "needs review" invoices

Possible cause: Confidence threshold too high (default: 70%)

Fix: Lower threshold to 60% using --threshold 60 flag. This will auto-assign more invoices but may reduce accuracy slightly.

Issue: Non-IT vendor not flagged

Possible cause: Keyword not in detection list

Fix: Contact development team to add the keyword to admin_alerts.py detection rules.

Issue: Batch processing is slow

Expected time: ~30ms per invoice (1000 invoices in ~30 seconds)

If slower: Contact development team to enable parallel processing for large batches.

Best Practices

  • Review Auto-Assigned Invoices: Spot-check 5-10% of auto-assigned codes to ensure accuracy
  • Provide Detailed Corrections: When correcting, add a reason in the "Reason" column to help the system learn
  • Approve Admin Alerts Promptly: Non-IT vendors will keep triggering alerts until approved
  • Run Regularly: Process invoices monthly or quarterly to keep data current
  • Monitor Metrics: Track auto-assignment rate over time - it should improve as the system learns
  • Feedback Loop: The more corrections you provide, the smarter the system becomes
