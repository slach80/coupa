Core Architecture

The classification system uses a multi-stage pipeline approach with confidence scoring at each stage. The engine prioritizes historical data, then commodity/description analysis, with non-IT vendor detection as a safety check.

Classification Algorithm (8 Stages)

Each stage attempts to classify the invoice line. If successful with sufficient confidence, the process stops. Otherwise, it proceeds to the next stage.

Stage 1: Manual Overrides (100% Confidence)

Check database for admin corrections. If an administrator has previously corrected this invoice or supplier, that decision takes absolute precedence.

Stage 2: Exact Historical Match (95% Confidence)

Lookup supplier in validated mappings table. Exact string match against supplier_activity_mappings where admin_approved=true.

Stage 3: Fuzzy Supplier Match (60-90% Confidence)

Use RapidFuzz library to find similar supplier names (threshold ≥80%). Handles typos, variations, and extra whitespace. Keeps suffixes like "(AZURE) 7954" intact for accurate matching.

Stage 4: Commodity Field Match (50-80% Confidence)

Extract keywords from commodity field and match against activity code definitions. For example, "Computer Hardware" → WP100, "Cloud Services" → IN100.

Stage 5: Description Field Match (40-70% Confidence)

Keyword analysis on invoice line description. Lower confidence than commodity since descriptions are more varied and noisy.

Stage 6: Historical Pattern Match (50-75% Confidence)

Analyze supplier patterns from prior months. If supplier consistently received same code (even if not validated), suggest with moderate confidence.

Stage 7: Non-IT Vendor Detection (30% Confidence)

Keyword-based detection of non-IT vendors (CLEANING, CASINO, BENEFITS, etc.). Always flags for admin approval. May suggest appropriate SC code.

Stage 8: No Match Found (0% Confidence)

New supplier with no historical data or keyword matches. Flagged for manual review.

Confidence Scoring

Confidence Range System Action User Experience
90-100% Auto-assign (exact match, manual override) Invoice automatically classified, appears in "Auto-Assigned" report
70-89% Auto-assign (strong fuzzy/commodity match) Invoice automatically classified, spot-check recommended
50-69% Provide top 3 suggestions Appears in "Needs Review" with alternatives, user selects best
20-49% Provide suggestions + flag Appears in "Needs Review", low confidence warning shown
0-19% Flag for manual review No suggestions, requires manual classification

Confidence Modifiers

  • New Supplier: -20% penalty (never seen before)
  • Multiple Historical Codes: -10% penalty (supplier has inconsistent past classifications)
  • Non-IT Vendor: Auto-flag for admin regardless of confidence

Database Schema

Table Purpose Key Fields

Data Flow

INPUT: Coupa Invoice Line └─→ Normalize supplier name └─→ Check manual_overrides (Stage 1) └─→ Check exact historical match (Stage 2) └─→ Try fuzzy supplier match (Stage 3) └─→ Try commodity match (Stage 4) └─→ Try description match (Stage 5) └─→ Try historical pattern (Stage 6) └─→ Check non-IT keywords (Stage 7) └─→ No match found (Stage 8) DECISION TREE: IF confidence ≥ 70%: ├─→ Auto-assign activity code ├─→ Store in classification_history └─→ Optionally sync to Coupa API ELSE IF confidence 20-70%: ├─→ Store top 3 suggestions ├─→ Flag for review (needs_review = TRUE) └─→ Add to review report ELSE IF confidence < 20%: ├─→ Flag for manual review └─→ Add to manual review queue IF admin_alert = TRUE: └─→ Add to admin_alerts table └─→ Await admin approval FEEDBACK LOOP: Admin corrects classification └─→ Store in manual_overrides └─→ IF same supplier corrected 3+ times: └─→ Update supplier_activity_mappings └─→ Future invoices auto-assign with 95% confidence

Key Features

🔍 Fuzzy Matching

RapidFuzz library handles supplier name variations, typos, and formatting differences while preserving important suffixes for accurate matching.

📊 Multi-Signal Analysis

Combines supplier name + commodity code + description + historical patterns for robust classification across diverse invoice types.

🔄 Learning Loop

Manual corrections automatically update supplier mappings. System accuracy improves over time as more corrections are provided.

⚠️ Non-IT Detection

Prevents incorrect IT activity code assignment to non-IT vendors (cleaning, legal, benefits) with keyword-based detection and admin approval workflow.

📋 Complete Audit Trail

Every classification decision is logged with method used, confidence score, suggestions considered, and timestamp. Supports compliance and quality review.

⚡ Scalable Pipeline

Processes thousands of invoices efficiently. Early-exit optimization stops at first confident match, avoiding unnecessary computation.

Design Philosophy

Confidence over Coverage: The system prioritizes accuracy over auto-assignment rate. A 60% auto-assignment rate with 95% accuracy is better than 80% auto-assignment with 85% accuracy.

Human-in-the-Loop: The system is designed to assist, not replace, human decision-making. Low-confidence cases are flagged for review rather than guessed.

Continuous Improvement: The feedback loop ensures the system learns from corrections and becomes more accurate over time without manual retraining.
