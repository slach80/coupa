Phase-by-Phase Breakdown

Week 1: Database & Data Loading

Goal: Establish database foundation and load historical data

  • Create SQLite schema (5 tables)
  • Load Activity Codes_Active sheet → activity_codes table
  • Parse pipe-delimited supplier names → supplier_activity_mappings
  • Write database unit tests
  • Create data loading script

Deliverables:

  • ✓ classification/database.py with SQLAlchemy models
  • ✓ scripts/load_activity_codes.py
  • ✓ Populated SQLite database with 59 activity codes
  • ✓ 40+ supplier-to-code historical mappings

Week 2-3: Classification Engine

Goal: Build core classification logic with all matchers

  • Implement fuzzy matcher (RapidFuzz)
  • Build keyword matcher (commodity + description)
  • Create non-IT vendor detector with admin alert logic
  • Implement confidence scoring algorithms
  • Build classification engine orchestrator
  • Write comprehensive unit tests (≥90% coverage)

Deliverables:

  • ✓ classification/engine.py - main classification logic
  • ✓ classification/matchers/fuzzy_matcher.py
  • ✓ classification/matchers/keyword_matcher.py
  • ✓ classification/admin_alerts.py
  • ✓ classification/confidence.py
  • ✓ Full unit test suite

Week 3: Batch Processor & CLI

Goal: Process SAMPLE.xlsx and generate review reports

  • Build batch processor for Excel files
  • Create CLI interface for manual runs
  • Implement review report generator (Excel output)
  • Process SAMPLE.xlsx (all 4 months: 15,072 invoice lines)
  • Store results in classification_history table
  • Generate admin alerts for non-IT vendors

Deliverables:

  • ✓ classification/batch/processor.py
  • ✓ scripts/classify_invoices.py CLI
  • ✓ classification/batch/reporter.py
  • ✓ Classified SAMPLE.xlsx with confidence scores
  • ✓ Review report Excel file (4 sheets)

Week 4: Testing & Iteration

Goal: Validate accuracy and fine-tune thresholds

  • Review auto-assigned codes (≥70% confidence)
  • Validate admin alerts (non-IT vendors)
  • Test suggestion quality (20-70% confidence band)
  • Fine-tune confidence thresholds
  • Add missing keyword rules
  • Performance testing (batch 3,768 lines)

Success Metrics:

  • ✓ Auto-assignment rate ≥60%
  • ✓ Accuracy on validation sample ≥95%
  • ✓ Admin alert precision ≥80%
  • ✓ Top-3 suggestions contain correct code ≥85%
  • ✓ Batch processing time <4 hours for quarterly data

Week 5: Feedback Loop & Production Prep

Goal: Implement learning system and prepare for production

  • Build manual override handling
  • Implement admin approval workflow
  • Create feedback loop (corrections → mappings)
  • Re-run batch processor to validate improvements
  • Write user documentation
  • Prepare deployment package

Deliverables:

  • ✓ scripts/admin_review.py CLI tool
  • ✓ classification/feedback_loop.py
  • ✓ Updated supplier_activity_mappings from corrections
  • ✓ User guide documentation
  • ✓ Deployment instructions
  • ✓ Production-ready package

Resource Requirements

Role Time Commitment Responsibilities
Backend Developer Full-time (5 weeks) Classification engine, database, testing
Finance SME Part-time (5-10 hrs/week) Validate classifications, review reports, define rules
DevOps Engineer Part-time (Week 5) Deployment, scheduling (if batch job needed)
QA Engineer Part-time (Week 4) Test accuracy, edge cases, performance testing

Milestones & Gates

  • Week 1 Gate: Database populated with 59 activity codes and 40+ historical mappings
  • Week 3 Gate: SAMPLE.xlsx successfully processed with ≥50% auto-assignment rate
  • Week 4 Gate: Validation sample shows ≥90% accuracy on auto-assigned codes
  • Week 5 Gate: Feedback loop demonstrates accuracy improvement from corrections

Risks & Mitigation

  • Risk: Classification accuracy below target (≥95%)
    Mitigation: Week 4 dedicated to fine-tuning. Can adjust confidence thresholds to increase precision at cost of auto-assignment rate.
  • Risk: Performance issues with large batches
    Mitigation: Use pandas for efficient DataFrame operations. Add multiprocessing if needed.
  • Risk: Non-IT vendors not flagged correctly
    Mitigation: Conservative keyword list with admin alert workflow. Can expand keywords iteratively.
  • Risk: Development delays
    Mitigation: Phased approach allows early value (CLI) even if MCP integration is delayed.

Post-Launch Plan

Month 1-3: Monitoring & Tuning

  • Weekly review of admin alerts and manual corrections
  • Track accuracy metrics on production data
  • Expand keyword rules based on misclassifications
  • Fine-tune confidence thresholds if needed

Month 3-6: Optimization

  • Implement MCP tool integration (if not in initial phase)
  • Add scheduled batch job automation
  • Build admin approval UI (if demand exists)
  • Performance optimization for >10K invoice batches

Month 6+: Enhancement

  • LLM-based classification for ambiguous cases (optional)
  • Real-time classification via Coupa webhooks
  • Vendor scoring system (phase 2 of project)
  • Predictive analytics (spend forecasting, vendor risk)
