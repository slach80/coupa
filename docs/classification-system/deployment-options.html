Deployment Scenarios Overview

The Activity Code Classification System can be deployed in multiple configurations depending on operational needs, technical constraints, and integration requirements. Each approach offers different trade-offs in complexity, real-time capability, and maintenance overhead.

Scenario Complexity Real-Time User Interface Best For
Option 1: Standalone Batch Script Low No CLI Only Quick wins, MVP, historical data processing
Option 2: MCP Tool Integration Medium Yes MCP Clients Real-time classification, Coupa workflow integration
Option 3: Scheduled Batch Job Low Near Real-Time CLI Only Regular processing, overnight runs, cron-based
Option 4: Web Application High Yes Modern Web UI Enterprise deployment, multi-user, admin dashboard
Option 5: Hybrid (Recommended) Medium Yes Multiple Flexibility, batch + on-demand + web, production-ready

New: For detailed web application architecture, see Web Application Architecture →

Option 1: Standalone Batch Script

SIMPLEST

Standalone Python Script

Self-contained classification engine with CLI interface. Runs independently of Coupa MCP server. Processes Excel files or CSV exports directly.

┌─────────────────────────────────────────────────────┐ │ EXCEL FILE (SAMPLE.xlsx) │ │ - Nov2025_Inv, Dec2025_Inv, etc. │ └──────────────────┬──────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────────────────────┐ │ STANDALONE CLASSIFICATION SCRIPT │ │ ┌───────────────────────────────────────────────┐ │ │ │ classify_invoices.py │ │ │ │ - Load Excel/CSV │ │ │ │ - Run classification engine │ │ │ │ - Generate review reports │ │ │ │ - Export updated files │ │ │ └───────────────────────────────────────────────┘ │ └──────────────────┬──────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────────────────────┐ │ SQLITE DATABASE │ │ - supplier_activity_mappings │ │ - classification_history │ │ - admin_alerts │ └─────────────────────────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────────────────────┐ │ OUTPUT FILES │ │ - classified_invoices_YYYYMMDD.xlsx │ │ - review_report_YYYYMMDD.xlsx │ └─────────────────────────────────────────────────────┘

✓ Advantages

  • Simplest to implement (2-3 weeks)
  • No dependencies on Coupa MCP infrastructure
  • Easy to test and debug
  • Portable - runs on any machine with Python
  • Quick wins processing historical data
  • Minimal deployment complexity

✗ Limitations

  • No real-time classification
  • Manual file export from Coupa required
  • No automated Coupa integration
  • Requires manual upload of classified codes
  • Less suitable for production workflows
  • Data synchronization challenges

Usage Example

# Run classification on SAMPLE.xlsx $ python classify_invoices.py --input SAMPLE.xlsx --output classified_output.xlsx # Generate review report $ python classify_invoices.py --input SAMPLE.xlsx --report-only --threshold 70 # Process specific sheet $ python classify_invoices.py --input SAMPLE.xlsx --sheet "Nov2025_Inv"

Best Use Cases

  • MVP / Proof of concept to demonstrate value
  • Historical data cleanup (backfill past quarters)
  • One-time migration projects
  • Testing classification rules before production deployment

Option 2: MCP Tool Integration

REAL-TIME

Native Coupa MCP Tools

Extend existing Coupa MCP server with classification tools. Enables real-time classification as invoices are created in Coupa.

┌─────────────────────────────────────────────────────┐ │ COUPA PROCUREMENT API │ │ - Invoices, POs, Suppliers, Requisitions │ └──────────────────┬──────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────────────────────┐ │ COUPA MCP SERVER (Extended) │ │ ┌───────────────────────────────────────────────┐ │ │ │ Existing Tools: │ │ │ │ - coupa_list_invoices │ │ │ │ - coupa_get_invoice │ │ │ │ - coupa_search_invoices │ │ │ └───────────────────────────────────────────────┘ │ │ ┌───────────────────────────────────────────────┐ │ │ │ NEW Classification Tools: │ │ │ │ - coupa_classify_invoice │ │ │ │ - coupa_batch_classify_invoices │ │ │ │ - coupa_get_classification_suggestions │ │ │ │ - coupa_approve_classification │ │ │ │ - coupa_get_admin_alerts │ │ │ └───────────────────────────────────────────────┘ │ └──────────────────┬──────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────────────────────┐ │ CLASSIFICATION ENGINE (Python Module) │ │ - Fuzzy matching, confidence scoring │ │ - Non-IT detection, admin alerts │ └──────────────────┬──────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────────────────────┐ │ SQLITE DATABASE │ └─────────────────────────────────────────────────────┘

✓ Advantages

  • Real-time classification on invoice creation
  • Native integration with Coupa workflows
  • No manual file exports required
  • Seamless MCP tool discoverability
  • Unified interface with existing Coupa tools
  • Can update Coupa invoices directly (if API permits)

✗ Limitations

  • More complex development (3-4 weeks)
  • Tight coupling to MCP server lifecycle
  • Requires MCP server deployment knowledge
  • Debugging more complex (network, API)
  • Performance constraints (synchronous calls)
  • Limited batch processing efficiency

New MCP Tools

# Classify a single invoice coupa_classify_invoice(invoice_id: str, line_number: int) → Returns: {activity_code, confidence, method, needs_review, suggestions} # Batch classify multiple invoices coupa_batch_classify_invoices(invoice_ids: List[str]) → Returns: {results: [...], summary: {...}} # Get classification suggestions for manual review coupa_get_classification_suggestions(invoice_id: str, line_number: int) → Returns: {top_3_suggestions, confidence_scores, rationale} # Approve admin alert (non-IT vendor) coupa_approve_classification(invoice_id: str, activity_code: str) → Updates supplier_activity_mappings with admin approval # List admin alerts coupa_get_admin_alerts(status: "pending" | "resolved") → Returns: {alerts: [...]}

Best Use Cases

  • Real-time classification as invoices enter Coupa
  • Workflow automation (classify on invoice approval)
  • Interactive classification within MCP client (Claude Desktop, etc.)
  • API-first organizations with strong MCP adoption

Option 3: Scheduled Batch Job

AUTOMATED

Cron-Based Scheduled Processing

Automated batch job runs on schedule (nightly, weekly). Pulls invoices via MCP, classifies, stores results in database. No manual intervention required.

┌─────────────────────────────────────────────────────┐ │ CRON SCHEDULER (Linux/Windows Task) │ │ - Daily: 2:00 AM │ │ - Weekly: Sunday 3:00 AM │ └──────────────────┬──────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────────────────────┐ │ BATCH CLASSIFICATION JOB │ │ ┌───────────────────────────────────────────────┐ │ │ │ 1. Pull new invoices via coupa_list_invoices│ │ │ │ 2. Filter unclassified (missing activity code)│ │ │ 3. Run classification engine │ │ │ │ 4. Store results in database │ │ │ │ 5. Generate email report to finance team │ │ │ │ 6. Log completion status │ │ │ └───────────────────────────────────────────────┘ │ └──────────────────┬──────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────────────────────┐ │ COUPA MCP SERVER │ │ - coupa_list_invoices (fetch unclassified) │ │ - coupa_update_invoice (write back codes) │ └─────────────────────────────────────────────────────┘

✓ Advantages

  • Fully automated - no manual triggers
  • Near real-time (within 24 hours)
  • Efficient batch processing
  • Scheduled during off-peak hours
  • Email notifications for review
  • Low operational overhead

✗ Limitations

  • Delay between invoice creation and classification
  • Requires job scheduling infrastructure
  • Error handling complexity (retry logic)
  • Monitoring and alerting needed
  • Not suitable for urgent classifications
  • Potential data sync issues on failures

Implementation Example

# Crontab entry (run daily at 2 AM) 0 2 * * * /usr/bin/python3 /app/coupa/scripts/batch_classify.py --mode incremental # Script logic 1. Fetch invoices created/modified in last 24 hours 2. Filter invoices with missing activity codes 3. Run classification engine 4. Store results with confidence scores 5. Generate review report (CSV/Excel) 6. Email finance team with summary: - X invoices auto-assigned (≥70%) - Y invoices need review (<70%) - Z admin alerts (non-IT vendors)

Best Use Cases

  • Regular processing of new invoices (daily/weekly)
  • Organizations with established batch processing workflows
  • Compliance-driven environments (overnight processing windows)
  • When real-time classification is not required

Option 4: Hybrid Approach (Recommended)

RECOMMENDED

Flexible Hybrid Architecture

Combines standalone batch processing with optional MCP integration. Provides maximum flexibility for different use cases while maintaining simplicity.

┌─────────────────────────────────────────────────────┐ │ CLASSIFICATION ENGINE (Core Library) │ │ - Fuzzy matching, confidence scoring │ │ - Database management, admin alerts │ │ - Pydantic models, Pandera validation │ └──────────────────┬──────────────────────────────────┘ │ ┌─────────┴─────────┬─────────────────┐ │ │ │ ▼ ▼ ▼ ┌──────────────────┐ ┌──────────────┐ ┌───────────────┐ │ CLI Script │ │ MCP Tools │ │ Batch Job │ │ (Manual Run) │ │ (Real-Time) │ │ (Scheduled) │ └──────────────────┘ └──────────────┘ └───────────────┘ │ │ │ └───────────────────┴──────────────────┘ │ ▼ ┌─────────────────┐ │ SQLite Database│ └─────────────────┘

✓ Advantages

  • Maximum flexibility - choose deployment per use case
  • Start simple (CLI) → expand to MCP/batch later
  • Reusable core classification engine
  • Support both ad-hoc and automated workflows
  • Easy testing (CLI) and production (MCP/batch)
  • Gradual adoption path

✗ Limitations

  • Slightly more upfront design work
  • Need to maintain multiple interfaces
  • Documentation for 3 usage patterns
  • Testing complexity (3 entry points)

Architecture Components

1. Core Classification Engine

Shared Python library with all classification logic:

src/classification/ ├── engine.py # Main classification logic ├── matchers/ # Fuzzy, keyword, historical matchers ├── database.py # SQLAlchemy models & queries ├── models.py # Pydantic schemas ├── confidence.py # Confidence scoring algorithms └── admin_alerts.py # Non-IT vendor detection

2. CLI Interface (Standalone)

# Process Excel file $ python scripts/classify_invoices.py --input SAMPLE.xlsx # Generate review report only $ python scripts/classify_invoices.py --report-only --threshold 70 # Classify specific invoices $ python scripts/classify_invoices.py --invoice-ids "12345,12346,12347"

3. MCP Tools (Real-Time)

# New MCP tools in src/coupa_mcp/tools/classification.py - coupa_classify_invoice() - coupa_batch_classify_invoices() - coupa_get_classification_suggestions() - coupa_approve_classification() - coupa_get_admin_alerts()

4. Scheduled Batch Job

# Cron entry 0 2 * * * /usr/bin/python3 /app/scripts/batch_classify.py --mode incremental # Uses MCP to fetch invoices, classification engine to process

Phased Rollout Strategy

  • Phase 1 (Week 1-3): Build core engine + CLI script → Process SAMPLE.xlsx for validation
  • Phase 2 (Week 4): Add MCP tools → Enable real-time classification for testing
  • Phase 3 (Week 5): Implement scheduled batch job → Automate daily processing
  • Phase 4 (Ongoing): Fine-tune based on usage patterns → Deprecate unused interfaces

Best Use Cases

  • Organizations unsure of final deployment model
  • Need both historical backfill (CLI) and ongoing automation (batch/MCP)
  • Want to demonstrate value quickly (CLI) before committing to infrastructure
  • Teams with varying technical sophistication (finance uses reports, IT uses MCP)

Option 5: Web Application

ENTERPRISE

Full-Stack Web Application

Modern web app with React frontend, FastAPI backend, PostgreSQL database, and direct Coupa API integration. Provides rich UI for finance teams with dashboards, admin workflows, and real-time collaboration.

✓ Advantages

  • Modern, intuitive web interface
  • Multi-user collaboration with RBAC
  • Real-time dashboards and analytics
  • Direct two-way sync with Coupa API
  • Mobile-responsive design
  • Rich notification system
  • Complete audit trail
  • Enterprise SSO integration

✗ Limitations

  • Longer development time (10 weeks)
  • Higher infrastructure costs
  • More complex deployment
  • Requires dedicated hosting
  • Ongoing maintenance overhead
  • User training required

Architecture Highlights

  • Frontend: React 18 + Redux Toolkit + Material-UI
  • Backend: FastAPI + SQLAlchemy + Celery
  • Database: PostgreSQL (primary) + Redis (cache/queue)
  • API Integration: Async Coupa API client with rate limiting
  • Auth: JWT + OAuth2 + SSO support

Recommendation: Phased Hybrid Approach

Phase 1 (5 weeks): Build core classification engine + CLI/batch interfaces for immediate value.

Phase 2 (10 weeks, optional): Add web application based on Phase 1 feedback and adoption.

This approach provides:

  • Immediate ROI via CLI processing of historical data (Week 3)
  • Validation of classification accuracy before web UI investment
  • MCP integration for real-time use cases (Week 4)
  • Automated batch job for ongoing processing (Week 5)
  • User feedback informs web UI design (if needed)
  • Flexibility to stop after Phase 1 if sufficient
  • Full web app with enterprise features (Phase 2)

Implementation Considerations

Infrastructure Requirements

  • Standalone CLI: Python 3.10+, SQLite (local file), minimal
  • MCP Integration: Coupa MCP server deployment, network access to Coupa API
  • Scheduled Batch: Cron/Task Scheduler, persistent storage, email SMTP
  • Hybrid: All of the above (opt-in per component)

Security Considerations

  • Coupa API Keys: Secure storage via environment variables or secrets manager
  • Database Access: File permissions on SQLite, or PostgreSQL with auth
  • Review Reports: Sensitive supplier/invoice data - limit access
  • Admin Alerts: Email notifications should be encrypted (TLS)

Monitoring & Logging

  • Classification Metrics: Track auto-assignment rate, confidence distribution
  • Error Handling: Log API failures, database errors, invalid data
  • Performance: Monitor batch processing time, API latency
  • Alerting: Notify on batch job failures, low confidence trends
